# 队长发车

你是当前的队长,现在整轮发车前讨论已经结束,轮到你**必须正式发车**了——提议一支执行本任务的队伍。队伍人数必须正好等于本任务要求的人数(见游戏状态)。你可以把自己算进去。根据当前公开信息、任务人数和你的当前目标选择队伍。

7 人局当前任务人数依次是 2-3-3-4-4。早期低信息时,队长自带是常规且容易负责的选择,可以用来建立基准;中后期应根据公开票型、任务成功/失败、失败牌数量和玩家前后矛盾收缩队伍。

发车不是只为立刻成功,也用于制造下一轮能讨论的票型和任务结果。公开上常见的稳健结构是自己+叠票位+发言/票型较好的玩家;非常规发车也可以成立,但公开理由必须能被普通玩家接受。

如果桌面已经有 2 个任务成功,发车目标先看能否直接拿第三蓝。若上一轮成功队伍人数正好等于当前任务人数,且没有公开矛盾或失败记录,原车复跑是必须优先讨论的方案;任何换新位/继续验证都需要解释为什么值得放弃直接胜利机会。

如果已经有 2 个任务失败(再失败一个或被打到强制轮就结束),你处在必须赢下这一车的赛点,且没有可复用的成功原车:优先发一支最干净的队形,把已经跳派、或公开给出队形、敢于负责的可信位/身份位带上车,少用纯试探车冒险。

低信息早期可以沿座位附近顺发或反方向组队,重点是建立基准,不要跳太远导致自己像开视野。常规稳健车可考虑自己 + 前面反复出现的叠票位 + 一个发言/票型较好的玩家;若两人互相强打,不要硬塞同车。

强制轮或后段发车时,优先自己 + 连续被认可的票型组合,少突然外撒无关位。非常规试探车可以观察外白/内反或验证叠票位;公开仍包装成“跑信息/建基准”。

如果当前还没有玩家公开发言、投票结果或任务结果,不要声称观察到某人的沉稳、发言风格、抢话习惯、投票模式、任务行为或矛盾。开局可以坦白说信息少,这是试探性组队、队长自荐、缩短责任链或建立基准。

在 `speech` 里用 1-2 句简要说明公开理由——但记住所有人都会听到,所以要往对自己有利的方向去包装。你不必如实说出真正的理由,可以公开说成“测试组合/建立基准/先跑结果”,把真实计划、怀疑对象或钓鱼意图写进私有 `memo`,但不要编造公开记录里没有发生过的事实。

请严格按以下 JSON 格式回复(队伍人数必须符合要求):

`{ "action": "propose_team", "team": ["C", "D", "E"], "speech": "我发 C-D-E,先测一个新组合,别第一轮就过度绑定。", "memo": "公开说测试组合;私下想看D的投票和任务牌压力反应。" }`
