# 队伍投票

队长发车了一支队伍(见游戏状态)。请投票**赞成**或**否决**。所有人投完后票面会同时公开,所以你的投票会变成公开信息——之后可能要为它辩解。

考虑:你信任这支队伍吗?为了把队长权交给下一个人,值得否决它吗?记住同一任务里 5 次发车被否决会直接把胜利送给邪恶方,所以否决并非没有代价。你可以用可选 `memo` 记录私下投票理由,不公开。

零信息的第一轮第一次发车,默认应赞成:开局没有任何公开事实能支撑反对,无理由否决只是把发车权往下递、白白消耗提案次数(同一任务 5 次否决直接判负),还会让你之后很难为这张反对票辩解。除非你能指出公开可见的理由——异常发车、矛盾发言、票型问题——否则首轮不要投反对。想表达谨慎就写进 `memo`,不要用反对票表达。

票型参考:你在车内赞成叫内白,车外反对叫外黑,通常信息量较低;车内反对叫内黑,车外赞成叫外白,更值得记录,但必须结合任务结果判断。不要为了显得会盘而投异常票;你的票之后会公开,要能用公开事实解释。

投票前做一次票型模拟:如果这车成功,谁会因为内白/外白被抬高;如果失败,新增上车位、内黑、外白谁最可疑。异常票可以投,但必须能用公开事实解释,并把真实目的写进 `memo`。

如果好人已经拿到 2 个成功(2-0 或 2-2),投票先看这车是否放弃了可直接三蓝的原车。上一轮成功原车人数正好满足当前任务时,“老带新/继续验身份”的替换车需要更强理由;除非原车已有公开矛盾,否则直接赢第三个任务的队伍优先级更高。

请严格回复以下 JSON 之一:

`{ "action": "vote", "approve": true, "memo": "可选私有投票理由" }`

`{ "action": "vote", "approve": false, "memo": "可选私有投票理由" }`
